# Refactor Safely

> 検証コマンド・生成物のパスは規約 `.claude/docs/dev-workflow-conventions.md` §1 の役割に
> 従う（既定値はそのまま例示）。別プロジェクトでは検証コマンドと生成物を読み替える。

外部から観測できる振る舞い（生成物・出力・分析結果の数値など）を変えないことが絶対条件。

1. リファクタ前に**全検証**を緑にする（規約 §1。既定例: pytest / ruff / mypy / `make build`）
2. リファクタ対象と「変えない契約」を明記する
   （例: `clustering(data)` の戻り dict のキー・型・丸め桁は不変）
3. 可能なら**ゴールデン比較**で守る: リファクタ前の生成物を退避し、後の生成物と差分0を確認
   - `make build` → site/data, site/fragments を一時コピー → リファクタ → 再build → `git diff` で差分確認
4. 小さなステップで変更し、各ステップで検証を回す
5. 仕様変更・機能追加を混ぜない（混ざったら別タスク・別 PR に分割）
6. 変更前後で生成物の差分が無い（または説明可能な差分のみ）ことを証跡として示す

## 禁止事項

- リファクタと称して数値・スキーマ・シードを変えない
- 「ついで」の機能追加をしない
