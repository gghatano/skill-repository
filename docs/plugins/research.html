research

研究レポート作成ライフサイクル

実験レポートの骨子設計から実験の計画・実行、レビュー、公開までを一式で進める研究リポジトリ向けのスキル群。役割ベースのスキル本体・規約・サブエージェントを同梱。

/plugin install research@gghatano-skills

何を入力して、何をして、何を得るか

入力
実験のテーマ・データと、対象リポジトリのレイアウト（役割→パスの規約）。
すること
レポート骨子の設計 → 実験の計画・実行 → 各種レビュー → 公開整合チェックを段階的に進める。
得られるもの
レビュー済みの実験レポートと、公開できる成果物一式。

含まれるスキル（9）

上の流れは、次のスキルを組み合わせて実現します。

  • doc-cycleドキュメント全体の整理・レビュー・対応サイクルを一括実行するオーケストレーター。report-review → repro-engineering-review → related-info-review → publish-check を順次呼び出し、横断的な指摘をまとめて報告する。実験の追記後・公開前・定期点検に使う。
  • experiment-plan実験計画の作成とレビュー。レポート骨子から抽出した実験一覧をもとに、既存研究に準拠した指標による公平な比較条件と、業務課題を反映した評価シナリオを計画書（docs/plans/）に落とす。実験の追加・比較条件の変更時にも使う。
  • experiment-run実験の実行と結果の記録。計画書（docs/plans/）に基づき、uv による固定環境でスクリプトを実行し、結果をレポート規約（実行条件の明記・図表・所見）に沿って整理する。実験の再実行・シード追加・新実験スクリプトの追加時にも使う。
  • publish-check公開サイトの整合確認。content/ の Markdown と htmls/ のビルド成果物、ビルダー（scripts/03_build_html.py）の PAGES テーブル、ナビ、README のリンクの整合を点検し、ビルドを実行して検証する。ページの追加・改名・公開前の最終確認に使う。
  • related-info-review関連情報の整理。参考文献・出典タグ・上流リポジトリ情報・手法解説ページの参考リンク・残課題（GitHub Issues）を点検し、リンク切れや対応漏れを整理する。文献追加、上流の更新追従、Issue 化の準備時にも使う。
  • report-review実験レポート（前提・結果・考察・課題）の整理とレビュー。レポート本体や追加実験の Markdown を、事実と推定の分離・数値の整合・統計的な断定の妥当性・アカデミック体裁・課題の追跡可能性の観点で点検し、軽微な問題は修正する。レポートの章立て直し、実験結果の追記、考察の見直し時にも使う。
  • report-skeletonレポート骨子の設計。実験を始める前にレポートの章立て・各章の主張・必要な実験と指標を骨子として固め、段階的に埋めるための TODO 付きスケルトンを content/ に用意する。新しい評価対象に着手するとき・レポートの再構成時に最初に使う。
  • repro-engineering-review再現手順とエンジニアリングノートのレビュー。setup/usage/reproduce/engineering-notes とスクリプト・依存固定・パッチの整合を点検し、clone-and-run で再現が成立するかを確認する。環境構築手順の変更、依存更新、スクリプト追加時にも使う。
  • research-cycle実験レポート作成のライフサイクル全体を統括するトップレベルのオーケストレーター。report-skeleton → experiment-plan → experiment-run（実験ごとに反復）→ doc-cycle を順に進め、現在地（stage・未実施の計画・残 TODO）を判定して次のプロセスへ誘導する。新しい評価対象への着手時、または「いま何をすべきか」の確認に使う。

実行手順の例

代表的な流れです。状況に応じて省略・前後してかまいません。

  1. 1/research:report-skeleton — レポートの骨子を設計する
  2. 2/research:experiment-plan → /research:experiment-run — 実験を計画して実行する
  3. 3/research:report-review / repro-engineering-review / related-info-review — 各観点でレビューする
  4. 4/research:publish-check — 公開前の整合を確認する（/research:research-cycle が全体の進行役）

同梱物（規約・エージェント）

導入後の調整

docs/documentation-conventions.md の §1「パスの読み替え（役割→既定パス）」表を、対象リポジトリの実際のレイアウトに合わせて調整してください。役割ベースのスキル本体は編集不要です。

writing プラグインの stop-ai-slop-jp を併せて導入していると、report-review が自動で連携します。

導入手順

  1. マーケットプレイスを追加（初回のみ）/plugin marketplace add gghatano/skill-repository
  2. このプラグインを導入 /plugin install research@gghatano-skills
  3. スキルを名前空間付きで呼び出す（例 /research:doc-cycle）
