generation

仕様駆動 生成・評価サイクル

仕様から成果物を作り、評価して直す汎用サイクル（input-prepare → spec-ingest → generation-design → generate → evaluate-refine）。骨格は領域非依存で、現在の中身は合成データ生成プロファイル。

/plugin install generation@gghatano-skills

何を入力して、何をして、何を得るか

入力
生成対象の仕様とサンプル（タスクディレクトリ `$ARGUMENTS` 配下の `input/`）。
すること
input-prepare → spec-ingest → generation-design → generate → evaluate-refine を回し、作って評価して直す。
得られるもの
仕様を満たす生成物（`output/`）と、忠実性・有用性・プライバシーの評価結果。

含まれるスキル（6）

上の流れは、次のスキルを組み合わせて実現します。

  • evaluate-refineタスクディレクトリ ($ARGUMENTS) の合成データを、仕様・サンプル・制約に照らして評価し、必要に応じて generator.py を修正する。evaluation_report.md と constraints_check.csv を出力する。
  • generateタスクディレクトリ ($ARGUMENTS) の generation_plan.md に基づき、src/generator.py を実装し、output/ に合成データを生成する。
  • generation-cycle合成データ生成パイプラインの PM (project manager) エージェントとして起動する。必要に応じて原資料から input/ を作成し、5 つの SKILL (input-prepare → spec-ingest → generation-design → generate → evaluate-refine) を順次サブエージェントに委譲し、各ステップの Acceptance Criteria 充足を確認しながら end-to-end で実行する。
  • generation-designタスクディレクトリ ($ARGUMENTS) の inferred_schema.json と constraint_plan.md をもとに、各列の生成方式を設計し work/generation_plan.md を作成する。
  • input-prepareタスクディレクトリ ($ARGUMENTS) の原データ・仕様書・業務ドキュメントを読み取り、合成データ生成パイプラインが参照する input/ 配下の table_definition、sample_data、data_spec.md、constraints.md を作成・更新する。
  • spec-ingestタスクディレクトリ ($ARGUMENTS) の input/ を読み取り、合成データ生成に必要な仕様を機械可読化する。work/inferred_schema.json と work/constraint_plan.md を作成する。

実行手順の例

代表的な流れです。状況に応じて省略・前後してかまいません。

  1. 1/generation:input-prepare — input/ に仕様とサンプルを揃える
  2. 2/generation:spec-ingest — 仕様を取り込んで生成要件に落とす
  3. 3/generation:generation-design — 生成方式（ルール／統計／モデル／差分プライバシー）を設計する
  4. 4/generation:generate — 生成物を output/ に作る
  5. 5/generation:evaluate-refine — 忠実性・有用性・プライバシーで評価して直す（/generation:generation-cycle で全体を自動運転も可）

同梱物（規約・エージェント）

docs/generation-cycle-conventions.md

導入後の調整

汎用の生成・評価サイクルです。骨格（フェーズ）は領域非依存で、docs/generation-cycle-conventions.md にプロファイルの考え方をまとめています。各スキルはタスクディレクトリ（$ARGUMENTS）配下の input/ work/ output/ src/ を読み書きします。別の生成領域を扱うときは、骨格・スキル名はそのままに、各フェーズの中身を差し替えたプロファイルを用意してください。

導入手順

  1. マーケットプレイスを追加（初回のみ）/plugin marketplace add gghatano/skill-repository
  2. このプラグインを導入 /plugin install generation@gghatano-skills
  3. スキルを名前空間付きで呼び出す（例 /generation:evaluate-refine）
