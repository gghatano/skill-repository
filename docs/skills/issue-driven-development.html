issue-driven-development

GitHub Issue 単位で実装作業を一貫管理する手順。

/dev:issue-driven-development
SKILL.md を見る ↗

概要

GitHub Issue を作業単位として branch/worktree/PR/検証記録を一貫管理する。実装作業を始めるときに必ず使う。

できること

  • Issue 本文を作業指示書として要約する
  • Issue 専用の branch/worktree を確認する
  • 既存パターンに合わせ実装しテストを追加する
  • 規約 §1 の検証コマンドを通しで実行する
  • PR 作成と Issue への検証記録・相互リンクを行う

使いどころ

すべての実装作業を始めるときに必ず使う。1 Issue = 1 branch = 1 worktree = 1 PR で管理する。

入出力・連携

入力は Issue 番号、出力は PR と Issue への検証結果コメント。spec-implementation / pr-from-issue や code-reviewer 等のサブエージェントと連携する。

このスキルを導入

プラグイン dev の一部です。

/plugin install dev@gghatano-skills
