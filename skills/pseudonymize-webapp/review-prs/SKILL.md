name: review-prs
description: >
  openなPRを並列サブエージェントでレビューし、結果をGitHubコメントとして投稿する。
  `/review-prs` で起動。引数なしで全open PR、PR番号指定で個別レビュー可。
---

## 動作

1. `gh pr list --state open` でopen PRを取得
2. 各PRに対してサブエージェントを並列起動しレビュー実行
3. レビュー結果を `gh pr review <番号> --comment` でPRにコメント投稿

## 引数

- 引数なし: 全open PRをレビュー
- PR番号（スペース区切りで複数可）: 指定PRのみレビュー
  - 例: `/review-prs 57 58`

## サブエージェント起動時のプロンプトテンプレート

各PRのサブエージェントには以下の情報を渡すこと:

```
PR #<番号>「<タイトル>」(branch: <ブランチ名>) をレビューしてほしい。

手順:
1. `gh pr view <番号>` でPR概要確認
2. `gh pr diff <番号>` で差分全体を確認
3. 以下の観点でレビュー

レビュー観点:
- コード品質: 可読性、命名、重複、不要なコード
- テスト: 網羅性、エッジケース、テストの実効性
- セキュリティ: インジェクション、XSS、OWASP Top 10
- アーキテクチャ: 既存パターンとの一貫性、state管理への影響
- パフォーマンス: N+1、不要な全件処理、メモリ使用
- htmx干渉: hx-target/hx-swap/DOM構造への影響（該当する場合）

出力フォーマット（日本語）:
- 総合評価: Approve / Request Changes / Comment
- 良い点（箇条書き）
- 指摘事項（重要度付き: 重要/中程度/軽微）
- セキュリティ評価
- まとめ
```

## コメント投稿フォーマット

```
gh pr review <番号> --comment --body "$(cat <<'EOF'
## コードレビュー結果: <Approve等の判定>

### 良い点
- ...

### 指摘
1. **[重要度] タイトル**: 説明

### セキュリティ
問題なし / 指摘あり

🤖 Generated with [Claude Code](https://claude.com/claude-code)
EOF
)"
```

## 注意事項

- サブエージェントは最大5並列（CLAUDE.mdの並列ルールに従う）
- open PRが0件の場合は「open PRなし」と報告して終了
- レビュー完了後、全PRの結果サマリをユーザに表示する
- コメント投稿前にユーザ確認は不要（レビューコメントは非破壊的）
